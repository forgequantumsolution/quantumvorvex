## ADDON 1 — Smart Pricing & Revenue Management

### Frontend

Add a **Revenue Engine** card to `Settings → Tax & Pricing` tab, below the existing GST configuration:

**Dynamic Pricing Rules (inline editable table)**

| Rule | Threshold | Adjustment | Active |
|------|-----------|------------|--------|
| High Demand Surge | Occupancy ≥ 80% | +15% rate | toggle |
| Long Stay Discount | Stay ≥ 30 days | −10% rate | toggle |
| Extended Stay Discount | Stay ≥ 90 days | −20% rate | toggle |
| Early Bird Discount | Booked 15+ days ahead | −8% rate | toggle |

- Each row is inline-editable (threshold number, adjustment %, active toggle)
- `+ Add Rule` button appends a new row
- `Save Rules` persists via `PUT /api/v1/settings/pricing-rules`

**Competitor Rate Benchmarking panel** (below pricing rules):
- Manual entry table: Competitor Name | Room Type | Their Daily Rate ₹ | Your Rate ₹ | Δ %
- Small Recharts `<BarChart>` comparing your rates vs competitors per room type (gold bar = yours, grey bar = competitor)
- `+ Add Competitor` button

**Dashboard Integration:**

Add a **"Revenue Opportunity"** alert card to the existing Notifications panel:
- Triggers when any room has been vacant for ≥ 7 consecutive days
- Format: `⚠️ Room [number] vacant 9 days — consider a discount or promotion`
- One alert per qualifying room, dismissable
- Auto-generated by the daily cron job (see Backend)

**Room Rate Display in Check-In Step 2:**
- When a room is selected, show computed rate with applicable discount/surge badge
- Example: `₹2,500/day` → `₹2,125/day ✦ Long Stay −15%`
- Tooltip explains which rule applied

### Backend

```
GET  /api/v1/settings/pricing-rules         — list all rules
PUT  /api/v1/settings/pricing-rules         — bulk upsert rules
POST /api/v1/pricing/compute                — body: { roomId, checkIn, checkOut, stayType } → returns { baseRate, appliedRule, finalRate, breakdown }
GET  /api/v1/reports/vacancy-alerts         — rooms vacant ≥ N days (N from settings)
```

**Pricing compute logic (util: `utils/pricingEngine.js`):**
1. Fetch base rate from room type
2. Check occupancy %: if `(occupied / total) >= 0.80` and surge rule active → apply surge
3. Check stay length: compare against discount thresholds (longest qualifying discount wins)
4. Check booking lead time: if `checkIn - today >= 15 days` → apply early bird
5. Rules are mutually exclusive (surge overrides discount); configurable in settings

### Database

```prisma
model PricingRule {
  id          String  @id @default(cuid())
  name        String
  triggerType String  // occupancy | stay_length | lead_time
  threshold   Float
  adjustment  Float   // positive = increase %, negative = decrease %
  active      Boolean @default(true)
  createdAt   DateTime @default(now())
}

model CompetitorRate {
  id           String @id @default(cuid())
  name         String
  roomType     String
  theirRate    Float
  recordedDate DateTime @default(now())
}
```

**GST slab computation (`utils/gstEngine.js`):**
```javascript
function computeGST(roomRate, gstConfig) {
  if (!gstConfig.autoSlab) return gstConfig.defaultRate;
  if (roomRate < 1000) return 0;
  if (roomRate <= 7500) return 12;
  return 18;
}
```
